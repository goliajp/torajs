## Why this baseline matters

L1 vision item #1 is **perf 远超 bun/nodejs · metal/llvm-level**. The current
state at P4.2 closer is the integration point where:

1. **Substrate state** — Layer-1 fully Rust (pool/rc/ucd/anyvalue/throw),
   Layer-2 fully Rust (str/num/bigint), Layer-3 partially Rust (arr/dynobj
   landed; map/set/weak/promise/regex/fetch/cycle/date still in C).
2. **Coming changes (Phase 1 remainder)** — port runtime_map.c (971 LOC),
   runtime_weakmap/set/ref (~671 LOC), runtime_cycle.c (510 LOC),
   runtime_promise.c (1165 LOC), runtime_regex.c (3059 LOC, biggest single
   substrate), runtime_fetch.c (189 LOC), runtime_date.c (589 LOC).
3. **Expected Phase 1 close state** — `torajs-runtime` becomes a thin
   glue/dispatch shim (libc_bridge.c 61 LOC + dispatch only); the 11-file
   C runtime collapses entirely to Rust sub-crates.

After Phase 1 close, re-run this exact bench corpus with the same
methodology + machine; compare row-by-row. Targets:
- **Neutral or faster** on every case (Rust ports must not regress).
- **Same artifact_bytes order** (~100-200 KB; no bloat).
- **Same compile_ms order** (~200-400 ms; no slowdown).

Polish (Phase 3) only kicks in AFTER Phase 2 (0 deps) ships clean —
per L1 vision the order is rewrite → defragment deps → polish, not
inter-leaved.
